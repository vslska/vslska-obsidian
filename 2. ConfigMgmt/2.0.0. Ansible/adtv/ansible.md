**Ansible** — это **инструмент автоматизации** , который позволяет:

- Управлять конфигурацией.
- Развертывать приложения.
- Выполнять оркестровку задач на множестве серверов.
- Делать всё это **без установки агентов** на удалённые машины (agentless).

Он использует **SSH** для подключения к серверам и выполняет команды **параллельно** , что делает его очень удобным и мощным.

Control  node (Master) -  (хост, узел) с которой запускается ansible
Managed node (Clien) - ( хост, узел) которыми управляешь через ansible

**Inventory** - файл где перечислены все узлы, обычно назвается hosts or inventory.ini
Пример: 
[hosts] - группа хостов, можно создавать большое количество групп, также создавать группы и объединять их в группы. (children)

**Ansible.cfg** - Основной конфигруационный файл Ansible. позволяет задавать глобальные настройки.
- Путь к inventory
- Логирование
- Где искать модули
- Пользователь по умолчанию и т.д.

Модули

Готовые "фукции", которые выполняют определенные действия
Например:
- Ping - проверяет связь между control node и managed node.
- Setup - выдача данных о сервере
- Shell -  зпуск комманмд облолочки на managed.
- Copy - копирует файлы. 
- Url - скачать фалы на managet a
- Apt/Yum - устанавливает пакеты.
- Service - управляет службами
- User - создает/удаляет пользователей.

 Ad-hoc комманды

Команды, которые ты запускаешь напрямую из терминала.  
Подходят для быстрого выполнения простых задач.

Пример:

ansible servers -m apt -a "name=vim state=present"

> Обновит пакет `vim` на всех хостах из группы `servers`.

Playbook

Это **YAML-файл** , в котором описано, какие задачи выполнять, на каких хостах, в какой последовательности.

Playbook — основной инструмент Ansible.  
Именно ими обычно пользуются для автоматизации.
