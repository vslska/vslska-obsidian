## Output values, или выходные значения

Выходные значения предоставляют информацию об инфраструктуре в командной строке и могут предоставлять информацию для использования другими конфигурациями Terraform.

**Выходные значения используются в:**

- **Корневом модуле** Для вывода определённых значений в CLI после запуска `terraform apply`.
- **Дочернем модуле** Для того, чтобы предоставить набор атрибутов своих ресурсов родительскому модулю. Подробнее об использовании модулей в Terraform вы узнаете в следующих темах.
Ресурсы, управляемые Terraform, экспортируют атрибуты. Их значения могут быть использованы в других частях конфигурации. Выходные значения — это способ предоставить пользователю вашего модуля экспортируемые атрибуты.
### Как объявлять выходные значения

Каждое выходное значение output value должно быть объявлено с помощью блока `output` и названия этого значения:
```
output "cluster_id" {
  description = "PostgreSQL cluster ID"
  value       = yandex_mdb_postgresql_cluster.this.id
}
```
По этому названию выходное значение будет отображено в CLI и доступно для дочернего модуля.
Аргумент `value` содержит выражение, которое будет возвращено. В примере выше это атрибут `id` экземпляра ресурса `this` ресурса `yandex_mdb_postgresql_cluster`.

Выходные значения отображаются только при применении вашего плана Terraform. Запуск команды `terraform plan` не вызывает отображение выходных значений.
В выводе CLI выходное значение отображается так:
```
Outputs:

cluster_id = "your-postgresql-cluster-id"
```
## Дополнительные аргументы

Блок `output` может включать дополнительные аргументы.
##### **Аргумент `description`**
Описывает выходные значения.

> [!NOTE] Важно
> Описание должно кратко объяснять, какое значение будет получено и каково его предназначение. Эта строка описания может быть включена в документацию о модуле, поэтому она должна быть написана с точки зрения пользователя модуля, а не его автора.
##### **Аргумент `sensitive`**
Ограничивает вывод интерфейса Terraform при использовании выходного значения. Объявление выходного значения с аргументом `sensitive = true` заменит его значение в выводе команд `terraform plan` и `terraform apply` на `(sensitive value)`.

Ограничивает вывод интерфейса Terraform при использовании выходного значения. Объявление выходного значения с аргументом `sensitive = true` заменит его значение в выводе команд `terraform plan` и `terraform apply` на `(sensitive value)`.

> [!NOTE] Важно
> Аргумент `depends_on` следует использовать только в крайнем случае. При его использовании всегда добавляйте комментарий с причиной его использования, чтобы помочь будущим инженерам понять цель дополнительной зависимости.
