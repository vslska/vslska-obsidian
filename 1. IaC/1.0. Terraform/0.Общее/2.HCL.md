2**HCL, или HashiCorp Configuration Language,** — это язык конфигурации для описания инфраструктуры как кода в Terraform.
# Синтаксис HCL
Для описания синтаксиса языка HCL можно выделить следующие основные элементы:

 **Декларативный подход** Означает, что вы описываете желаемое состояние вашей инфраструктуры, а не шаги для достижения этого состояния.
 
 **Блоки** Основной элемент синтаксиса HCL. Они используются для описания ресурсов, переменных, выходных значений и других компонентов конфигурации. Каждый блок начинается с открывающей фигурной скобки `{` и заканчивается закрывающей фигурной скобкой `}`
```hcl
data "yandex_compute_image" "ubuntu" {
  family = "ubuntu-2204-lts"
}
```

**Атрибуты** Внутри блоков используются атрибуты для определения параметров ресурсов или других элементов конфигурации. У атрибутов есть имя и значение, разделённые знаком равенства `=`.
```hcl
resource "yandex_compute_snapshot" "default" {
  name           = "test-snapshot"
  source_disk_id = "test_disk_id"
}
```

**Переменные** HCL позволяет определять переменные для использования в вашей конфигурации. Переменные могут быть определены в отдельных файлах или внутри конфигурационных файлов.
```hcl
variable "zone" {
  type    = string
  default = "ru-central1-a"
}   
```

**Встроенные функции Terraform** В HCL могут использоваться функции для динамического определения значений атрибутов и переменных, такие как арифметические операции и условные выражения.
Например, функция `timestamp`
```hcl
metadata = {
   ...
    date     = timestamp()
}
```

**Модули** позволяют организовывать и переиспользовать части конфигурации Terraform. Например, модуль `S3`:
```hcl
module "s3" {
  source      = "https://github.com/terraform-yc-modules/terraform-yc-s3"
  bucket_name = "unique-bucket-name"
}
```

**Выходные атрибуты** С помощью выходных атрибутов можно определять данные, которые будут доступны после применения конфигурации Terraform. Эти значения могут использоваться в других конфигурациях или скриптах.
```hcl
output "image_id" {
  description = "Image ID from data block"
  value       = data.yandex_compute_image.default.image_id
}
```
