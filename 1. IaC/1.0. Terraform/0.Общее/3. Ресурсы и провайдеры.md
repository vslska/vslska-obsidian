# Ресурсы и провайдеры
Ресурсы — это инфраструктурные объекты (виртуальные машины, хранилища данных, сетевые интерфейсы и т. д), которыми вы хотите управлять с помощью Terraform. У каждого ресурса есть собственный тип и набор атрибутов.

```hcl
resource "yandex_vpc_network" "default" {
  name = "vpc-name"
}
```

В этом примере:
- `"yandex_vpc_network"` — тип ресурса, в данном случае — VPC в облаке Yandex Cloud;
- `"default"` — локальное имя для данного экземпляра ресурса/блока;
- `name` — атрибут этого ресурса, определяющий имя VPC.

Провайдеры в Terraform определяют использование конкретных облачных платформ или других систем, с которыми Terraform взаимодействует для управления ресурсами. У каждого провайдера — собственная документация, где описаны поддерживаемые ресурсы и параметры. Перед использованием ресурса лучше свериться с документацией, например с [yandex_vpc_network](https://terraform-provider.yandexcloud.net/resources/vpc_network), которую мы использовали в примере выше.
### Пример синтаксиса использования провайдера в Terraform

```hcl
provider "yandex" {
  cloud_id                 = "cloud_id_here"
  folder_id                = "folder_id_here"
}
```
