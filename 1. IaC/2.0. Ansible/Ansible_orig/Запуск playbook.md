#### Пример запуска 
```
ansible-playbook test.yml --verbose
```

Используйте флаг --verbose при запуске вашего плейбука, чтобы увидеть подробный вывод как успешных, так и неудачных модулей.

#### Запуск плейбука в режиме проверки (check mode)
Режим проверки Ansible позволяет вам выполнить плейбук без внесения каких-либо изменений в ваши системы. Вы можете использовать режим проверки для тестирования плейбуков перед их реализацией в производственной среде.
Чтобы запустить плейбук в режиме проверки, вы можете передать флаг -C или --check команде ansible-playbook:
```
ansible-playbook --check playbook.yaml
```

#### ansible-pull
Если вы хотите инвертировать архитектуру Ansible так, чтобы узлы регистрировались в центральном месте, а не пересылали им конфигурацию, вы можете это сделать. Ansible-pull — это небольшой скрипт, который извлекает репозиторий инструкций по конфигурации из git, а затем запускает ansible-playbook для этого содержимого. Если предположить, что вы балансируете нагрузку на место оформления заказа, ansible-pull масштабируется практически бесконечно. Для получения подробной информации выполните ansible-pull --help.
