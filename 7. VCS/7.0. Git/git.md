##### Git Command

###### Основные команды Git

**Инициализация репозитория** Чтобы начать использовать Git в новом проекте, нужно инициализировать репозиторий в нужной директории:
```bash
git init
```

Эта команда создаст скрытый каталог `.git` в текущей директории, где будет храниться вся информация о репозитории.

**Просмотр состояния репозитория** Команда `git status` показывает текущее состояние репозитория, включая файлы, которые были изменены, добавлены или удалены.
```bash
git status
```

**Добавление файлов в индекс** Перед тем как сделать коммит, нужно добавить файлы в индекс (стейджинг). Это можно сделать с помощью команды `git add`.
```bash
git add <файл>
```

Или добавить все изменённые файлы ( . )
```bash
git add .
```

**Создание коммита** После добавления файлов в индекс можно создать коммит — сохранить текущее состояние репозитория.
```bash
git commit -m "Описание коммита"
```

**Просмотр истории коммитов** Команда `git log` показывает историю всех коммитов в репозитории.
```bash
 git log
```

Удаление объекта  из индекса
```bash
git rm --cached <каталог/файл>
```

**Работа с удалёнными репозиториями** Для работы с удалёнными репозиториями (например, на GitHub) используются команды `git remote`, `git push` и `git pull`.

Добавление удалённого репозитория:
```bash
 git remote add origin https://github.com/username/repository.git
```

Отправка изменений на удалённый репозиторий:
```bash
git push -u origin main # Для ветки main
git push -u origin master # Для ветки master
```

Получение изменений из удалённого репозитория:
```bash
git pull origin main # Для ветки main
git pull origin master # Для ветки master
```

**Ветвление и слияние** Ветвление позволяет работать над новыми функциями или исправлениями ошибок в отдельных ветках, не затрагивая основную ветку.

Создание новой ветки:
```bash
git branch new-feature
```

Переключение на другую ветку:
```bash
 git checkout new-feature
```

Создание и переключение на новую ветку за одну команду:
```bash
 git checkout -b new-feature
```

Слияние ветки в основную:
```bash
git checkout main # Переключаемся на основную ветку
git merge new-feature # Сливаем изменения из new-feature в main
```

**Удаление ветки** После того как ветка больше не нужна, её можно удалить.
```
git branch -d new-feature
```

**Отмена изменений** Если нужно отменить изменения в файле до последнего коммита:
```bash
git checkout -- <файл>
```

Если нужно отменить добавление файла в индекс:
```bash
git reset HEAD <файл>
```

**Использование `.gitignore`** Файл `.gitignore` позволяет исключить определённые файлы или директории из отслеживания Git. Например, временные файлы или логи.

Пример содержимого `.gitignore`:

```
*.log

*.tmp

*.exe

/build/
```

Глобльно изменить при создании локлаьного main (master по умолчанию)
```bash
git config --global init.defaultBranch main
```

Переименовать локльную ветку с master в main и т.п. 
```bash
git branch -m master main
```


Если ветки не привязаны ( локал и удаленная)
```bash
git push --set-upstream main main
```

##### Рабочий процесс с Git
**Начало работы** :

Инициализируй репозиторий в нужной директории:
` git init`

Добавь файлы в индекс и сделай первый коммит:
```bash
git add .
git commit -m "Первый коммит"
```

**Работа над проектом** :
Выполняй изменения в файлах.

Периодически добавляй изменения в индекс и делай коммиты:
```bash
git add .
git commit -m "Описание изменений"
```

**Создание веток для новых функций** :

Создай новую ветку для новой функциональности:bash
```bash
git checkout -b new-feature
```

Внеси изменения и сделай коммиты в этой ветке.

**Слияние веток** :

Когда закончишь работу над новой функцией, переключись на основную ветку и слей изменения:
```bash
git checkout main
git merge new-feature
```

Удали ненужную ветку:
```bash
git branch -d new-feature
```

**Отправка изменений на GitHub** :

Добавь удалённый репозиторий (если ещё не добавил)
```bash
git remote add origin https://github.com/username/repository.git
```

Отправь изменения на GitHub:
```bash
git push -u origin main
```
##### Советы для начинающих

- **Часто делай коммиты** : Чем чаще ты будешь делать коммиты, тем легче будет отслеживать изменения и откатывать их при необходимости.
- **Используй осмысленные сообщения к коммитам** : Сообщения к коммитам должны быть информативными и описывать, что было изменено.
- **Используй ветки** : Ветвление помогает разрабатывать новые функции и исправления ошибок без влияния на основную ветку.
- **Делай бэкапы** : Регулярно отправляй изменения на удалённый репозиторий, чтобы иметь актуальную копию проекта.
