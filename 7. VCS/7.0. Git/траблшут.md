
####  Ошибка авторизации

```bash
git:(master) git remote show origin https://github.com/vslska/vslskdotfiles.git
Username for 'https://github.com': vslska
Password for 'https://vslska@github.com':
remote: Invalid username or token. Password authentication is not supported for Git operations.
fatal: Authentication failed for 'https://github.com/vslska/vslskdotfiles.git/'
```

```bash
**«Password authentication is not supported for Git operations»**
```
Это значит:
- GitHub **заблокировал вход по паролю** (с 2021 года!)
- Нужно использовать **Personal Access Token (PAT)** **ИЛИ** **SSH**

 Так что **лучше всего перейти на SSH**
 Нужно, заместо  
```bash
 git remote add origin https://github.com/vslska/vslskdotfiles.git
```
нужно
```bash
git remote add origin git@github.com:vslska/vslskdotfiles.git
```

Изменяем подключение в локальном репозитории
```bash
git remote set-url origin git@github.com:vslska/vslskdotfiles.git
```

Проверить что SSH работает 
```bash
ssh -T git@github.com
```

#### Не пушит изменнеия


Проблема, когда данные уже имеются в удаленном репозитории но отличаются от локального.
Скачай изменения с GitHub и объедини** (рекомендуется)

```bash
# Подтяни изменения с GitHub (они войдут как отдельный коммит)
git pull --rebase origin main

# Теперь отправь
git push -u origin main

 `--rebase` — аккуратно "поставит" твои коммиты **поверх** тех, что на GitHub.

Если будут конфликты (маловероятно для dotfiles), Git скажет — тогда просто сохрани нужные файлы и сделай:

git add .
git rebase --continue
```


| Ошибка                                      | Причина                                                     | Решение                                                                                       |
| ------------------------------------------- | ----------------------------------------------------------- | --------------------------------------------------------------------------------------------- |
| `Could not resolve hostname github.com:...` | Попытка использовать Git-URL в `ssh`                        | Проверять SSH: `ssh -T git@github.com`  <br>Или использовать HTTPS: `https://github.com/...`  |
| `src refspec main does not match any`       | Нет коммитов в локальной ветке                              | 1. `git add .`  <br>2. `git commit -m "..."`  <br>3. _Потом_ `git push`                       |
| `remote origin already exists`              | Повторное добавление `origin`                               | `git remote -v` → посмотреть  <br>`git remote set-url origin НОВЫЙ_URL` — если нужно заменить |
| `Updates were rejected...`                  | Удалённая ветка впереди (например, README создан на GitHub) | `git pull --rebase origin main`  <br>→ потом `git push`                                       |
